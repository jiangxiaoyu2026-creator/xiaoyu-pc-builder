# 技能：生成硬件行情视频文案

## 触发条件
当用户要求生成"写一下今天的日报"、"硬件行情日报/周报文案"、"视频口播稿"、"行情汇总"等内容时，执行此技能。本文档为**最新版**强制约束规范。

## ⚠️ 踩过的坑与硬性要求（必须避免）

### 1. 数据源必须生产环境 API
- **接口**：`https://www.diyxx.com/api/external/market-report-data`
- **Header**：`X-API-Key: diyxx-ai-secret-key-2026`
- **优先级**：参数 `period=daily` 优先。如果 `totalItemChanged=0`，说明当日无变动，回退抓取 `weekly`。

### 2. 现价断层问题（最容易踩坑）
- 如果产品今天没变动（比如标杆 NV3），你**必须**去抓 `weekly` 甚至 `monthly` 的接口，拿到它最后一次变动的 `newPrice` 作为“现价”来播报，绝对不能只说“价格稳定”而不报错真实数字，更不准自己编！

### 3. 多品类数据概览必须包含
- **规定动作**：在内存、CPU、显卡这三个段落的开头，必须计算并播报：**“变动的 X 款产品中有 Y 款都在降/涨。整体降幅/涨幅 Z%，平均降价/涨价金额 W块。”** 这四个数据缺一不可，需通过代码对 `categories` 里面的品类详单进行数学合并运算得出。

### 4. CPU 月度走势对比机制
- 播报 CPU 单品变动时，必须查询 `monthly` 接口提取该单品的历史价格，在单品价格播报后加上**月度趋势判断**。（如："参考最近一个月，这颗CPU仍然是涨价状态" / "创下了历史最低"）。

[极值产品1]，降了/涨了 [金额] 块，目前价格 [现价]。
[极值产品2]，降了/涨了 [金额] 块，目前价格 [现价]。

顺便看一下颗粒厂商的股票，[SK海力士/三星/美光等当日或当周股市情况]，[结合产能/AI行情的一句预测，如：在AI强劲需求的带动下，预计高端内存颗粒产能依然紧俏]。

★ *注：新版已取消内存标杆强行播报机制。*

★ *注：硬盘仅保留 NV3 1T 作为标杆必报产品，如果无变动，一定要报展现价。*

## 必须要做的交叉验证
生成脚本后，要求必须输出一张 `Markdown` 表格，对脚本中提到的所有的【产品名、老价格、新价格、涨跌幅度】和线上 API 文件二次对比并出具验证结果。一旦验证对不上的，必须自动回去重写对应的句子。所有的查价和提取行为应使用 Python 脚本，并对每个品类的 sum, count 做运算。
